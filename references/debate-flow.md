# 辩论流程规范

本文档是辩论框架的核心流程规范，不依赖任何特定平台或工具。你可以将此流程用于人工辩论、AI 辅助辩论或全自动辩论场景。

## 辩论角色

| 角色 | 职责 |
|------|------|
| **正方** | 为辩题的一个立场辩护，负责提出论据、回应反方攻击 |
| **反方** | 为辩题的对立立场辩护，负责反驳正方论点、提出反论据 |
| **裁判/主持人** | 中立角色，管理流程、核查事实、评估逻辑、引导深入 |

## 辩题准备

一个良好的辩题应符合三个条件：
1. **有争议空间**：双方都有合理的论证基础，不是一边倒
2. **有事实基础**：争议可以通过事实和逻辑来推进，不纯属价值观分歧
3. **有收敛可能**：辩论能导向一个更清晰的认识，不是原地打转

### 辩题解析

裁判在接手辩题后，先明确：
- 核心命题是什么？
- 正方的默认立场是什么？反方是什么？
- 双方最可能的分歧点在哪里？

## 辩论流程规范

### Phase 0：准备

1. 裁判解析辩题，明确核心命题和争议焦点
2. 生成辩论框架概览（辩题类型、正反方核心立场、初始分歧点）
3. 如适用，准备客观数据简报作为共识基础

### Phase 1：立论

1. **正方立论**：提出核心论点、论据和论证链条
2. **反方立论**：先回应正方核心论点，再提出己方论点和论据
3. **裁判点评**：事实核查 + 逻辑评估 + 提炼核心分歧

### Phase 2：攻防（可重复多轮）

每一轮由三个环节组成：
1. **正方发言**：针对上一轮反方论点回应 + 推进
2. **反方发言**：针对正方本轮论点回应 + 推进
3. **裁判点评**：事实核查 + 逻辑评估 + 态势框架图

每轮结束，裁判展示指令菜单（继续/终止/新话题）。

### Phase 3：终结

1. 辩论历程回顾
2. 事实共识表
3. 持续争议表
4. 全景框架图
5. 裁判最终意见
6. 开放问题

## 裁判实操指南

### 事实核查

裁判的核心职责之一。每轮必须核查：
- 双方引用的事实/数据是否有来源
- 数据是否被断章取义
- 是否存在选择性引用

### 逻辑评估

常见的逻辑漏洞：
- **稻草人**：曲解对方论点再攻击
- **虚假二分**：把复杂问题简化为二选一
- **滑坡谬误**：从 A 到 Z 的链条中缺少中间步骤的论证
- **循环论证**：结论已经包含在前提中
- **诉诸权威**：用"谁说的"代替"说了什么"

### 深度推进技巧

当辩论在立论层面僵持时，裁判应引导进入以下深度层次：

**第一层 → 第二层：从"谁对谁错"到"对到什么程度"**
- 避免是非题，改问程度题："A 替代 B 的速度取决于什么条件？"

**第二层 → 第三层：从"替代"到"分工"**
- 反方最强框架是将"替代"重构为"分工"——证明两者是互补而非零和
- 正方如需取胜，必须证明边界在向上移动

**第三层 → 第四层：从"当前"到"加速度"**
- 正方最有力的论证是"替代不是线性的"——引用 S 曲线采纳模型
- 一旦越过临界点，行为会从"试用"跳变为"主用"

### 收敛信号

2-3 轮攻防通常足够暴露核心分歧。以下信号表明可以终止：
- 双方开始重复论点
- 核心分歧已清晰化为"对一个未知变量的不同判断"
- 裁判可以给出"双方都没有错，分歧取决于一个尚未被市场回答的问题"
