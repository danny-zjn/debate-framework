# 🗣️ Debate Framework — 结构化辩论引擎

> **当讨论陷入"各说各话"，你需要一个裁判。**

不是聊天，不是吵架，不是各自写一篇小作文然后不管对方说了什么。

这是一个**有裁判、有流程、有框架图**的结构化辩论系统。正反方基于事实和逻辑攻防，裁判查数据、挑逻辑漏洞、用 ASCII 框架图把散点争论收束到一张图上。

---

## 这能用来干什么？

| 场景 | 例子 |
|------|------|
| **投资决策辩论** | "这只股票现在值不值得买？" — 正方看多，反方看空，裁判拉数据简报 |
| **项目评审辩论** | "这个项目该不该投？" — 基于投决会材料，两队各执一方 |
| **赛道对比辩论** | "仿真路线 vs 真机路线，哪个是终局？" — 每个项目各一个代表 |
| **战略方向辩论** | "我们应该自研还是采购？" — 内部正反论证 |
| **概念思辨** | "AI 会替代人类智能吗？" — 纯逻辑和知识层面的交锋 |

---

## 一次辩论长什么样？

```
Phase 0：裁判准备
  ├── 解析辩题，明确核心分歧
  └── 拉取客观数据简报（如适用）

Phase 1：立论陈词
  ├── 正方：核心论点 + 论据 + 逻辑链
  ├── 反方：针对正方回应 + 己方论据
  └── 裁判点评：事实核查 + 逻辑评估 + 提炼分歧

Phase 2：回合制攻防（N轮）
  ├── 正方回应反方 → 反方回应正方
  └── 裁判点评 + ASCII框架图 + 指令菜单（可/止/新话题）

Phase 3：终结收敛
  ├── 事实共识表 / 持续争议表
  ├── 全景框架图
  └── 裁判最终意见 + 开放问题
```

### 裁判的 ASCII 框架图长这样（真实产出）：

```
          辩题: 仿真 vs 真机，谁是具身数据终局？
                │
        ┌───────┴───────┐
        │               │
    正方(仿真)       反方(真机)
        │               │
   "规模是唯一壁垒"   "真实性不可替代"
        │               │
  合成100万条 > 真实1万  真机数据有闭环价值
        │               │
   ┌────┴────┐      ┌───┴───┐
   │SOTA证据 │      │客户合同│
   │63%→89%  │      │真机占比│
   └─────────┘      └───────┘
        │               │
        └───────┬───────┘
                │
         核心分歧: Sim2Real Gap
         收敛速度——1年还是5年？
```

一眼看清楚：双方各在什么立场、用什么证据、核心分歧到底在哪。

## 快速开始

### 作为 Hermes Agent Skill 使用

```bash
git clone https://github.com/danny-zjn/debate-framework.git ~/.hermes/skills/creative/debate
```

然后在 Hermes 里说一句：

> "辩论一下：中际旭创当前价位值不值得投资？"

或：

> "这是投决会材料，正方支持投，反方反对投，来一场辩论。"

### 独立使用（不依赖 Hermes）

直接读 `references/debate-flow.md`，流程规范是平台无关的，手动执行或接入任何 LLM 都行。

## 谁应该关注这个工具？

- **投资人** — 投决会前内部正反论证，暴露盲区
- **分析师** — 结构化对比两个标的，比逐项列表更暴露分歧
- **创业者/产品经理** — 战略方向论证，"自研还是采购"这类决策
- **任何需要做复杂决策的人** — 防止群体迷思，让反对意见有空间

## 核心原则

| 原则 | 说明 |
|------|------|
| **求真 > 求胜** | 目标是澄清事实，不是压倒对方 |
| **基于事实** | 论据必须有来源，禁止编造数据 |
| **裁判中立** | 不参与立场，只管理流程和核查 |
| **回合制攻防** | 每轮必须回应对方上一轮的论点，不可各说各话 |
| **可视化收敛** | 每轮结束时用 ASCII 框架图展示当前分歧结构 |

## 依赖
