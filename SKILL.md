# 辩论框架

## 核心原则

- **求真 > 求胜**：辩论的目的是澄清事实和逻辑，不是压倒对方
- **基于事实和逻辑**：论据必须有来源，推理必须可追溯。禁止凭空编造数据
- **主持人/裁判中立**：主持人不参与辩论立场，只管理流程和事实核查
- **每轮回应式攻防**：反方必须针对正方上一轮的论点做出回应，反之亦然

## 使用方式

### 触发条件

用户提及以下任一意图时触发：
- "辩论" / "辩一辩" / "来一场辩论"
- 用户给出辩题，要求正反方辩论
- 用户发来一份资料/文档，要求正反方基于此辩论
- 用户说"帮我看看这个项目值不值得投"等隐含辩论需求

### 三种输入模式

| 模式 | 用户输入示例 | 说明 |
|------|-------------|------|
| 概念辩论 | "辩论：AI是否会替代人类智能？" | 抽象议题，正反方基于各自专业知识辩论 |
| 事物辩论 | "辩论：某股票当前价位值不值得投资？" | 具体标的，裁判先拉数据简报作为共识基础 |
| 文档辩论 | "这是投决会材料，正方支持投资，反方反对投资" | 基于用户提供的文档进行立场辩论 |
| 比较性投资辩论 | "同赛道竞品 A vs B，哪个更值得投？" | 同赛道竞品对比，双方各代表一个项目。详见 `references/comparative-debate-template.md` |

## 辩论流程

### Phase 0：裁判准备

用户给出辩题后，裁判做三件事：

1. **解析辩题**：明确核心命题、争议焦点
2. **生成辩论框架概览**：展示给用户
   - 辩题类型、正反方核心立场、初始分歧点
3. **数据准备**（如适用）：
   - 股票/公司类辩题：通过 akshare / 行情 API 等数据源拉取客观数据简报（见 references/data-sources.md）
   - 文档类辩题：阅读用户提供的文档，提取关键事实作为共识基础
   - 抽象概念：无需数据准备

### Phase 1：立论陈词

#### 正方立论

```
【正方】【立论陈词】

1. 核心论点：[一句话表述正方的核心立场]

2. 论据支持：[摆出关键事实/数据/逻辑依据]

3. 论证展开：[逐步推理，展示逻辑链条]

4. 总结：[重申立场，指出此立场成立的必要条件]
```

#### 反方立论

```
【反方】【立论陈词】

1. 核心论点：[一句话表述反方的核心立场——与正方对立]

2. 对正方核心论点的初步回应：[针对性指出正方论点的薄弱之处]

3. 论据支持：[摆出反方的关键事实/数据/逻辑依据]

4. 论证展开：[逐步推理，展示反方的逻辑链条]

5. 总结：[重申立场，指出正方论点的关键缺陷]
```

#### 裁判点评（立论阶段）

```
【裁判】【立论点评】

1. 事实核查：双方引用的事实/数据是否有误

2. 逻辑评估：双方论证的逻辑链条是否自洽

3. 核心分歧：提炼双方最根本的分歧点

4. 下一轮引导：[提出供下一轮深入探讨的方向]
```

#### 展示指令菜单

```
【裁判】：(指令：可 / 止 / 新话题：XXXX)
```

---

### Phase 2：回合制攻防（多轮）

每一轮按以下格式执行：

#### 正方发言（针对上一轮反方论点进行回应+推进）

```
【正方】【第N轮】

1. 针对性论点：[对反方上一轮核心论点的直接回应——认同/反驳/修正]

2. 论据：[引用事实/数据/逻辑——如果是新引用数据，需注明来源]

3. 论证：[从论据到论点的推理过程，必须回应反方上一轮的逻辑链]

4. 新论点（如有）：[如果上一轮反方论点打开了新的争议面，可以在此提出新论点]

5. 总结：[本回合的结论，清楚指出反方论点的缺陷或己方论点的强化]
```

#### 反方发言（针对正方本轮论点进行回应+推进）

格式同上，但"针对性论点"必须直接回应正方本轮发言。

#### 裁判点评

```
【裁判】【第N轮点评】

1. 事实核查：
   - ✅ 确认的事实/数据：[列出本轮双方一致认可的事实]
   - ⚠️ 有问题的数据：[列出需要更正或质疑的数据]
   - ❌ 错误数据/编造：[明确指出错误并给出正确数据]

2. 逻辑评估：
   - 正方逻辑链：[评估——可成立/有漏洞/不成立]
   - 反方逻辑链：[评估——可成立/有漏洞/不成立]
   - 本轮攻防胜负：[指出哪一方在本轮更占理，并给出理由]

3. 当前态势——框架图：
   [ASCII art 框架图——直观展示当前辩论的核心分歧结构]

4. 核心争议推进：[本轮辩论是否推进了对核心问题的理解？新的分歧点是什么？]
```

#### 展示指令菜单

```
【裁判】：(指令：可 / 止 / 新话题：XXXX)
```

---

### Phase 3：终结与收敛

用户发出 `止` 指令后：

#### 全局总结

```
【裁判】【辩论总结】

1. 辩论历程回顾：
   - 初始命题：[辩题]
   - 论战轮次：共 X 轮
   - 核心交锋点：[列出辩论全程的2-3个核心交锋点]

2. 事实共识：
   [双方达成一致的事实/数据列表——表格形式]

3. 持续争议：
   [双方未达成一致的争议点——表格形式：争议点 | 正方立场 | 反方立场 | 裁判评估]

4. 框架图：
   [ASCII art 知识网络——展示辩题的全景结构：所有论点、反论点、证据链及其关系]

5. 裁判最终意见：
   [基于事实和逻辑，裁判指出哪一方的论证更具说服力，以及理由]
   [同时指出双方各自最有力的论点和最薄弱的环节]

6. 开放问题：
   [讨论中暴露但未穷尽的方向——可作为后续新话题的起点]
```

### 新话题指令

用户发出 `新话题：XXXX` 时：

1. **裁判评估相关性**：
   - 如果新话题与当前辩论同属一个核心领域，是当前讨论的合理延伸或深化方向 → **批准**，pivot 到新子话题
   - 如果新话题与当前辩论无关或相关性较弱 → 给出裁判意见，说明为什么认为不适合作为当前辩论的子话题，并询问用户是否坚持

2. **批准后的操作**：
   - 基于当前已建立的框架和事实共识，提出针对新子话题的正反方立场
   - **轮次编号不重置**：继续从当前轮次编号递增（如第3轮后切入新话题，下一轮为第4轮）
   - 继续辩论流程

## 裁判行为准则

1. **中立**：不偏袒任何一方，不参与立场辩论
2. **事实核查**：对双方引用的数据，有义务核实并指出错误
3. **逻辑评估**：指出逻辑漏洞，但不替任何一方补充论证
4. **挖深不铺广**：引导辩论集中在核心争议上，不被旁支带偏
5. **数据主动查询**：当辩论需要数据支持时，主动获取客观数据，并在裁判点评中公布
6. **股票类辩题**：
   - 辩论开始前必须先准备**客观数据简报**，作为双方共识基础
   - 简报包含但不限于：营收趋势、利润趋势、PE/PB 历史分位、毛利率/净利率、ROE、负债率、同行对比
   - 详见 references/data-sources.md
7. **文档类辩题**：
   - 用户发送文档后，先阅读文档，提取关键事实和假设
   - 双方辩论必须基于文档中的事实，不得编造文档中不存在的信息
   - 裁判负责提醒双方回到文档事实
